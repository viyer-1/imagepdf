Privacy Policy

TL;DR: We don't collect your files or your data. Ever.

Our Core Privacy Promise

Your files never leave your computer. All PDF conversions happen locally on your device. This is an open-source project — you can verify the code yourself.

1. What We DON'T Collect

  • Your Files: We never see, access, upload, or store any files you convert. All processing happens locally.
  • File Metadata: We don't collect information about what files you convert or their content.
  • Usage Telemetry: We don't track how you use the application, which features you use, or how often you use them.
  • Analytics: No Google Analytics, no third-party tracking pixels, no "phone home" functionality.
  • Device Information: We don't collect system information, hardware IDs, or any machine fingerprinting.

2. No Accounts, No Tracking

The Image ↔ PDF Converter Open Source Edition requires no account, no login, and no internet connection to function. We have deliberately built it to be "offline-first".

  • No Database: We do not run a central database of user accounts or email addresses.
  • No Licensing: All features are unlocked by default. There is no license validation or server check.

3. How We Protect Your Privacy

3.1 Local Processing

  • All file conversions happen on your computer using local processing libraries (Pillow, ReportLab, PyMuPDF).
  • No internet connection required for any part of the conversion process.
  • Files are never uploaded to our servers or any third-party servers.

3.2 Automatic Metadata Stripping

  • All EXIF metadata (GPS coordinates, camera info, timestamps) is automatically removed from images before conversion.
  • This protects you from accidentally leaking sensitive information in converted PDFs.

3.3 Transparency

  • The source code is fully available for audit.
  • You can build the application from source to ensure the binary matches the code.
  • No hidden "telemetry" or "analytics" modules.

4. Support Communications

When you contact the project team (e.g., via GitHub Issues):

  • Email/GitHub info: We see your contact info only to provide assistance.
  • Diagnostic information: If you voluntarily share logs or screenshots for troubleshooting, we use them only to resolve your issue.

5. HIPAA & Regulatory Compliance

5.1 HIPAA (Healthcare)

Our application is HIPAA-friendly because all processing happens locally on your computer. We never see or store Protected Health Information (PHI). Healthcare organizations can use our software without needing a BAA because the software is a local-only tool.

5.2 GDPR (European Union)

We are inherently GDPR-compliant as we do not collect or process personal data from the application's users.

6. Changes to This Policy

As an open-source project, any changes to the application's privacy behavior would be visible in the source code history. This policy may be updated to reflect project changes.
